Hosting

Where large datasets should live

The website stores metadata and links. Large files should be uploaded to a dataset platform or institutional storage.

Platform Choice

Choose a host according to access, audience, and citation needs.

ModelScope

Recommended for China-accessible public distribution. Use Git and Git LFS for large binary data, and keep the repository readable with README.md, info.json, and a documented data/ directory.

Kaggle

Recommended for international AI and data-science users. Include Kaggle's dataset-metadata.json, project info.json, and documented data files. Cluster CLI upload may require proxy configuration.

Zenodo

Recommended when a standalone dataset DOI is needed. Use it for archival releases and citeable snapshots.

Minimum Package

Every hosted dataset should include these files.

README.md
info.json
data/
checksums.sha256
load_example.py  # required for solver-specific binary formats

Short Upload Guide

Keep platform-specific files separate from scientific metadata.

Kaggle Package

Use kaggle datasets init to create dataset-metadata.json. The dataset id must match the Kaggle username and slug. The scientific metadata still belongs in info.json.

ModelScope Package

Create a dataset repository on ModelScope, clone it with Git, track large CFD files with Git LFS, then commit README.md, info.json, data/, and auxiliary scripts.

Large Binary Data

For raw binary fields such as Gvol.000001, V.000001, or P.000001, include a loading script and document the binary layout explicitly.

Binary CFD Files

Custom or extensionless files need explicit reading metadata.

For files such as Gvol.000001, V.000001, or other solver-specific binary outputs, the metadata should specify header size, dtype, payload shape, vector-component order, reshape order, grid association, and field location. A loading example is required so external users can verify that the data are readable.
