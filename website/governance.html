Governance

Automated precheck, human acceptance

The catalogue does not certify scientific perfection. It checks that datasets are understandable, citable, licensed, and practically usable, using automation for routine checks and maintainers for final review.

Review Model

Automation accelerates intake, but does not approve records.

The dataset workflow follows the practical pattern used by mature scientific data portals such as BLASTNet: contributors provide a standardized request, automated checks verify basic structure, and maintainers decide whether the record is complete enough to publish.

Automated Triage

Submission issues receive intake labels and reviewer notifications so new requests are less likely to be missed.

Metadata Precheck

Pull requests validate catalogue JSON records for required fields, access links, file format, and domain metadata.

Maintainer Review

Reviewers verify variables, units, grid, phase convention, file paths, license, citation, and practical readability.

Publication Decision

Maintainers mark accepted records as catalogue, community, benchmark, or archived after the automated and manual checks are resolved.

Human Checklist

Maintainers review the parts automation cannot judge.

  • Scientific scope matches multiphase-flow simulation, experiment, or AI-ready derived data.
  • Data access is public or restrictions are stated clearly.
  • License and citation terms are compatible with reuse.
  • Custom binary files include enough information to read at least one sample.
  • Benchmark submissions include stable splits, metrics, and baseline expectations.
