Data Standard

Metadata first, format flexible.

The catalogue accepts multiple scientific data formats, but every dataset must make variables, grid, units, file paths, and physical assumptions unambiguous.

Core Principle

Do not force one file format. Require clear meaning.

Multiphase-flow datasets may use HDF5, NPZ, VTK, EnSight, DAT files, or solver-specific binary outputs. The catalogue does not require a unified storage format, but every dataset must provide enough metadata for an external user to understand the variables, reconstruct the grid, load the files, and cite the source correctly.

Minimum Record

Required information for simulation datasets

Dataset Identity

Title, short description, contributors, contact, version, public access link, license, and DOI or citation.

Flow Fields

Velocity, pressure, and one interface or phase-indicator variable, with names, units, shapes, dtype, and field location.

Grid and Geometry

Grid dimensions, physical extent, coordinate order, cell- or node-centered storage, and grid files when needed.

Physics and Numerics

Configuration, phase convention, boundary conditions, initial conditions, solver or method, material parameters, and dimensionless numbers.

Variables

Complete CFD flow-field data should include three core roles.

For two-phase and multiphase CFD datasets, the minimum recommended fields are velocity, pressure, and an interface representation. Common names include U, V, velocity, u/v/w for velocity; P, p, or pressure for pressure; and alpha, Gvol, VOF, or phi for the interface. The phase convention must be stated, for example alpha = 1 for liquid and alpha = 0 for gas.

Parameters and Grid

Store case settings in metadata unless they vary as fields.

Fixed Parameters

Density, viscosity, surface tension, gravity, Reynolds number, Weber number, Ohnesorge number, and similar settings should normally be written in info.json.

Uniform Grids

Separate coordinate files are not required if dimensions, physical bounds, spacing, coordinate order, and field location are fully specified.

Complex Grids

Non-uniform, curvilinear, moving, adaptive, or unstructured grids must include the corresponding grid, geometry, or coordinate files.

Custom Binary Files

Extensionless or solver-specific binary files must document header bytes, dtype, payload shape, component order, reshape order, and include a loading example.

Metadata Template

Use info.json with global and local sections.

The global section describes the dataset as a whole. The local section maps each case, sample, or snapshot to its time value and file paths. Time is recommended when available, but a dataset with one or a few snapshots can still be useful if the physical state and file organization are clear. 

{
  "schema_version": "0.1.0",
  "dataset_type": "multiphase_flow_simulation",
  "global": {
    "dataset_id": "owner/dataset-slug",
    "title": "Dataset title",
    "description": "One paragraph dataset description.",
    "format": "hdf5 | npz | vtk | custom binary | ...",
    "snapshots": 1,
    "variables": [
      {
        "name": "alpha",
        "role": "phase_indicator",
        "units": "1",
        "shape": ["nx", "ny", "nz"],
        "dtype": "float32",
        "location": "cell_centered",
        "convention": "alpha=1 liquid, alpha=0 gas"
      }
    ],
    "grid": {
      "type": "uniform_cartesian",
      "cell_dims": [256, 256, 256],
      "domain_bounds_xyz": [[0, 1], [0, 1], [0, 1]],
      "coordinate_order": "x, y, z",
      "field_location": "cell_centered"
    },
    "physics": {
      "flow_type": "two_phase_flow",
      "configuration": "droplet breakup in HIT",
      "physical_parameters": {},
      "dimensionless_numbers": {},
      "boundary_conditions": "",
      "initial_conditions": ""
    },
    "contributors": [],
    "license": "CC BY 4.0",
    "doi": "",
    "known_limitations": []
  },
  "local": [
    {
      "snapshot_id": "000001",
      "time": {"value": null, "units": ""},
      "files": {
        "alpha": "data/alpha/alpha.000001",
        "u": "data/u/u.000001",

      }

}
