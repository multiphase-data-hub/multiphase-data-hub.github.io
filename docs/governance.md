# Governance

The project should be lightweight at the beginning and stricter only when the
community starts contributing.

## Maintainer Roles

- Catalogue maintainer: reviews metadata records and website changes.
- Scientific reviewer: checks whether the physics description is sufficient.
- Infrastructure maintainer: checks links, schemas, and automated validation.

One person can hold multiple roles during the early phase.

## Review Policy

Maintainers do not need to certify that every dataset is scientifically perfect.
They should verify that the record honestly documents:

- what the data contains;
- how it was produced;
- how it can be loaded;
- what license allows;
- what limitations are known.

## License Policy

Every dataset must choose an explicit data license. Recommended options:

- CC BY 4.0 when attribution is required;
- CC0 when maximum reuse is intended;
- ODC-BY for database-style releases;
- a custom institutional license only when unavoidable.

Code in this repository can use a permissive license such as MIT or BSD-3-Clause.

## Citation Policy

Each dataset page should show:

- dataset citation;
- related paper citation if available;
- software citation if solver code is citable;
- version and release date.
