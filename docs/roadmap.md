# Roadmap

## Phase 0: Internal Seed Release

- Finalize project name and GitHub organization.
- Choose repository license.
- Assign a data license for the first seed dataset.
- Upload the seed dataset archive to a persistent host.
- Fill DOI, checksum, authors, and citation fields.
- Add a minimal Python loader.

## Phase 1: Public Website

- Publish the static site via GitHub Pages.
- Add dataset search and filters by method, flow type, data format, and AI task.
- Add contribution instructions and a submission template.
- Invite trusted collaborators to review the first version.

## Phase 2: Community Contributions

- Create a dataset submission issue template.
- Add JSON schema validation in CI.
- Add link checking in CI.
- Start a curated list of benchmark tasks and baseline metrics.
- Invite datasets from nearby communities: atomization, droplets, bubbles,
  sprays, boiling, cavitation, and interface reconstruction.

## Phase 3: Research Infrastructure

- Provide loaders for HDF5, Zarr, NPZ, VTK, and EnSight.
- Publish benchmark notebooks.
- Add DOI-backed versioned releases.
- Build a small steering group across institutions.
- Prepare a short data descriptor or community paper.
